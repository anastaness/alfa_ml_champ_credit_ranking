# Подбор лучшего кредита — хакатон Альфа-Банка

Решение задачи ранжирования кредитных предложений.

## О хакатоне

- **Название:** Хакатон Альфа-Банк: Подбери лучший кредит
- **Организатор:** Альфа-Банк
- **Задача:** Ранжирование кредитных предложений (Learning to Rank)
- **Метрика:** NDCG@5 × 100

## Описание задачи

Банк формирует для клиента десятки кредитных предложений с разными параметрами. Необходимо ранжировать их так, чтобы предложения с наибольшей вероятностью отклика оказались в топе выдачи.

**Цель:** построить модель, которая для каждой заявки (`app_id`) сортирует предложения по убыванию вероятности сделки.

**Особенности:**
- 37 817 клиентов, в среднем 22 оффера на заявку
- Сильный дисбаланс: 4.48% сделок
- 100% клиентов в трейне имеют хотя бы одну сделку

> Примечание: после завершения соревнований был произведен пересчет метрики, 
> поэтому далее в описании приводятся значения до/после изменений.

## Результат

**NDCG@5 = 88.85/91.50**

| Этап | Офлайн (GroupKFold) | Онлайн (до / после) |
|------|---------------------|----------------------|
| LGBMClassifier (baseline) | 90.85 | 90.98 / 88.04 |
| LGBMRanker (lambdarank) | 91.10 | 91.36 / 88.71 |
| + Новые признаки (32 фичи) + ансамбль | 91.05 | **91.50 / 88.85** |


*Офлайн — средний NDCG@5 на GroupKFold (5 фолдов) по app_id.*
*Онлайн до — результат на лидерборде во время соревнования.*
*Онлайн после — результат пересчёта организаторами по уточнённой формуле.*

## Шаги решения

### Этап 1. EDA и отбор признаков

- **Целевая переменная:** 4.48% сделок, у 99.8% клиентов ровно одна сделка
- **variant_no=1** — доминирующий вариант: конверсия 24% (средняя 4.5%), есть у каждого клиента
- **need_2ndfl=Y** снижает конверсию до 0.26% (против 4.76% без справки)
- **ncl** (ожидаемые потери) положительно связан с конверсией: более рискованные кредиты доступнее
- **channel=Internet** конвертится на 0.8 п.п. лучше, чем Front
- **pil1mtrx_offer** проверен на data leakage: признак присутствует в тесте (2.70% vs 2.99%), 33.6% сделок без флага → не утечка, включён в модель
- **features_small.pq** (280 признаков БКИ): 82% пропусков при JOIN → не используются

### Этап 2. Feature Engineering (32 признака)

| Группа | Кол-во | Примеры |
|--------|--------|---------|
| Параметры оффера | 9 | variant_no, rate, term, limit, ncl, eva_perc |
| Бюрократия и канал | 7 | verif_need, need_2ndfl, verif_compl, channel, pil1mtrx_offer, req_loan_amount, req_term |
| Matching (оффер vs запрос) | 6 | diff_amount, abs_diff_term, ratio_amount, ratio_term |
| Basket (бизнес-правила) | 5 | has_desiredparams, has_minannuity, has_maxlimit, has_noverifdocmaxlimit, basket_rules_count |
| Комбинированные | 5 | rate/ncl, limit/term, req/limit, bureaucracy_score, term×rate |

### Этап 3. Моделирование

- **LGBMRanker** с `lambdarank` — оптимизирует NDCG напрямую, дал +0.38 / +0.67 к бейзлайну
- **Валидация:** GroupKFold (5 фолдов) по `app_id`
- **Гиперпараметры:** n_estimators=500, max_depth=6, num_leaves=32, reg_lambda=0.1, reg_alpha=0.1
- **Ансамбль:** 3 × LGBMRanker с разными seed (42, 123, 456)
- **Блендинг:** 0.5×seed42 + 0.3×seed123 + 0.2×seed456
- **Нормализация:** MinMax → [0, 1]

### Эксперименты (не вошли в финальную версию решения)

| Эксперимент | Офлайн NDCG | Причина отказа |
|-------------|-------------|----------------|
| CTR признаки (target encoding) | ~91.07 | LightGBM сам выделяет категории |
| CatBoost Ranker | 90.77 | Слабее LGBM на 0.33 |
| XGBoost Ranker | 90.77 | Слабее LGBM, требователен к формату qid |
| Клиентские признаки (features_small) | 91.02 | 82% пропусков создают шум |
| Ансамбль LGBM+CatBoost+XGBoost | ~91.33 | Хуже, чем ансамбль из трёх копий LGBMRanker |
| Топ-20 признаков (отбор по важности) | 80.90 | Потеря полезных фич |

## Запуск проекта

### 1. Установить Python

Требуется Python 3.10 или выше.

### 2. Клонировать репозиторий

```bash
git clone https://github.com/anastaness/alfa_ml_champ_credit_ranking.git
cd alfa_ml_champ_credit_ranking
```

### 3. Создать виртуальное окружение

```bash
python -m venv .venv
.venv\Scripts\activate      # Windows
source .venv/bin/activate   # Linux/macOS
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Подготовить данные

Файлы `train_dataset_small.pq`, `test_dataset_small.pq`, `features_small.pq` предоставлены организаторами хакатона. Поместите их в папку `data/`.

### 6. Запустить ноутбуки

```bash
jupyter notebook
```

1. `01_EDA_and_Feature_Selection.ipynb` — анализ данных и отбор признаков
2. `02_Feature_Engineering_and_Training.ipynb` — создание признаков, обучение модели, генерация сабмита
